## 4. ISS Tracking Data Flow

ELARA tracks the ISS in real-time, fetching Two-Line Element sets (TLE) and propagating orbits on a 3D Earth Globe.

Parallel Fetching: To ensure high availability, the system attempts to fetch TLE data from multiple mirrors simultaneously using Promise.any(). The first successful response is used; if all mirrors fail, an emergency fallback TLE is loaded.

## 7. File-by-File Analysis

### Core Logic (src/lib)

- skyScore.ts: The mathematical core for visibility. Implements the 40/35/25 weighting system.
- tleFetcher.ts: A robust fallback-oriented orbital data engine. Uses Promise.any() to fetch from 4+ sources in parallel.
- explanationService.ts: The AI adapter. It routes raw event titles to Google Gemini 1.5 Flash and caches the 2-sentence summaries.
- astronomy.ts: Astronomical utility functions.
- weather.ts: Integration with Open-Meteo for real-time cloud cover data.

### UI Components (src/components)

- EarthGlobe.tsx: A Three.js based 3D earth.
- SkyMap.tsx: A high-performance 2D Canvas component.
- SkyScoreRing.tsx: A Framer Motion powered HUD element.
- ISSTracker.tsx: Mission Control dashboard.
- StarField.tsx: Immersive Three.js background.

## 8. Feature Deep-Dive

### Stellar Scope (Sky Map)
The Stellar Scope is not an image; it's a dynamic coordinate transformation engine. It converts RA/Dec coordinates into local X/Y positions in real-time.

### AI-Enhanced Events
Uses Google Gemini to transform technical launch data into human-readable descriptions.

## 11. Maintenance & Troubleshooting

- TLE Sources: Parallel fetching via Promise.any().
- 429 Errors: Sequential AI processing to respect free-tier limits.
- ISS Position Latency: Multiple mirror sources.

---

## 12. Known Issues & Workarounds

- satellite.js WASM Warnings: Non-breakingWebpack issues.
- TLE Sources Timeout (Local): Fallback to hardcoded TLE.
- PWA Disabled in Dev: Intended behavior.
- Vercel Hobby Cron Limitation: 1/day limit.
- Google OAuth on New Domains: Whitelist authorized origins.

## 13. Vercel Deployment

### vercel.json configuration:
{
  "installCommand": "npm install --legacy-peer-deps",
  "crons": [
    {
      "path": "/api/cron/refresh-tle",
      "schedule": "0 0 * * *"
    },
    {
      "path": "/api/cron/check-events",
      "schedule": "0 12 * * *"
    }
  ]
}

### Deploy command:
npx vercel --prod

### After first deploy:
1. Environment Variable setup in Vercel UI.
2. Add NEXT_PUBLIC_APP_URL.
3. Redeploy.
4. Whitelist Domain in Firebase & Google Cloud.

### First-time Firestore setup:
Visit: https://your-url.vercel.app/api/seed
Required to initialize light pollution zones.
